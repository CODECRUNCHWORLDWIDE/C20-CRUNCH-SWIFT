# Week 19 — Challenges

The exercises drill basics. **Challenges stretch you.** This one takes 90–150 minutes and produces something you can commit to your portfolio and point at in an interview: a live watch complication driven by the shared core, plus a parity matrix that audits the share/adapt line across all five surfaces.

## Index

1. **[Challenge 1 — A watch complication + a five-platform parity matrix](challenge-01-watch-complication-and-parity.md)** — build a watchOS complication (a WidgetKit widget in the accessory families) that shows the live note count from the shared `NotesCore`, wire it to reload when notes change, and then produce a feature × platform parity matrix proving the shared core behaves identically across iPhone, iPad, Mac, Watch, and Vision while each shell fits its surface. You deliver the complication code, screenshots on the watch face, and the matrix with an honest "what each platform should and shouldn't do" column. (~120 min)

Challenges are optional. If you skip them, you can still pass the week. If you do this one, you'll be measurably ahead — "I shipped a watch complication driven by the same core as the phone, and here's the parity matrix showing the line held across five platforms" is exactly the concrete multi-platform competence that lands in code reviews and interviews. This is also direct preparation for the capstone's **multi-platform parity** rubric line (worth 15 points), which is this challenge scaled up to the full app.
