# Week 1 — Challenges

The exercises drill basics. **Challenges stretch you.** Each one takes 60–120 minutes and produces something you can commit to your portfolio.

## Index

1. **[Challenge 1 — Multi-file `wordfreq` with `--min-count`](challenge-01-wordfreq-multi-file.md)** — extend the mini-project's `wordfreq` CLI to accept multiple input files, merge their frequency counts, and add a `--min-count` flag, while keeping the Markdown table output and all tests green on both Linux and macOS. (~90 min)

Challenges are optional. If you skip them, you can still pass the week. If you do them, you'll be measurably ahead of someone who didn't — and the argument-parsing and dictionary-merge patterns here reappear in Week 3 (the link-checker CLI flags) and Week 5 (Vapor environment configuration).

> **Order of operations.** Do the [mini-project](../mini-project/README.md) first. The challenge assumes you already have a working `wordfreq` package with a `WordFreqCore` library target and a passing Swift Testing suite. The challenge *extends* that package; it does not start from scratch.
